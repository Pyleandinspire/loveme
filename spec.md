## 1. 项目概述

### 1.1 项目名称
**LoveMe - AI恋爱模拟器**

### 1.2 游戏简介
一款基于AI对话驱动的纯前端恋爱模拟游戏，使用 Flutter 框架开发。玩家通过与AI角色进行语音/文本对话，根据对话内容触发不同反馈、增减好感度，并在特定好感度阶段解锁特殊场景和剧情。同时包含多种趣味小游戏，增强与AI角色的互动体验。

### 1.3 核心玩法（升级版本）
- **无限沙盒对话**：玩家与AI角色的每一次对话都拥有独立的蝴蝶效应，不再有固定的剧情走向。
- **好感度推进机制**：玩家通过与AI角色进行文本/语音对话，根据对话内容触发不同的反馈和好感度变化。好感度从0开始，随着玩家的选择和互动逐步提升。
- **全动态里程碑事件（Dynamic Milestone Events）**：当好感度达到25、50、75、100等节点时，系统由"AI剧情导演"根据本局游戏中独特的分层记忆和聊天上下文历史，实时动态生成量身定制的专属剧情、抉择分支及多结局走向。
- **AI动态CG渲染（Procedural CG Generation）**：在里程碑事件或特殊结局触发时，大模型在返回文本的同时输出高精确度的画面描述词（Prompt），前端无缝并发调用图像生成引擎（如DALL-E 3）进行即时渲染，生成独一无二的全屏CG插画，并将其固化下载至本地，永不失效。
- **多周目平行宇宙机制**：引入周目（Playthrough）计数器。每一次开启新游戏都是一个全新的平行时空，由于记忆起点的不同，即使在相同的好感度节点，解锁的剧情过程、获得的CG图片以及最终的结局也完全不同。
- **趣味小游戏**：与AI角色对战，胜负影响好感度
- **永恒结局触发**：当好感度达到1000时，游戏进入最终结局阶段。系统将触发最终结局CG生成，由AI剧情导演根据玩家整个游戏历程生成专属的永恒结局画面和对话，标志着这段恋情的圆满完成。结局画面将自动保存至CG收藏夹，供玩家回顾。
- **AI角色形象自定义**：玩家可以在创建/选择角色时上传参考图片，AI将根据图片内容生成符合玩家预期的角色外貌描述，用于动态CG和头像生成。

### 1.4 AI服务方案（本地/单机双引擎版）

本游戏采用纯前端架构，不内置API Key，由玩家在设置页面自行配置。系统通过"文本生成"+"图像生成"+"多模态分析"三引擎共同驱动：

| 服务商/引擎 | 核心对话与剧情导演模型 | 对应支持的图像生成模型（动态CG） | 多模态图片分析模型 | 纯前端运行与调用机制 |
| :--- | :--- | :--- | :--- | :--- |
| **OpenAI** | `gpt-4o` / `gpt-4-turbo` | `dall-e-3` | `gpt-4o` / `gpt-4-turbo` | 前端通过标准REST API并发请求，获取文本后立刻异步下载图片二进制流 |
| **Claude** | Claude 3 Opus/Sonnet | DALL-E 3（需配置OpenAI Key） | Claude 3 Opus/Sonnet | 前端通过标准REST API并发请求 |
| **自定义/第三方代理** | 支持标准OpenAI格式的兼容模型 | Stable Diffusion API / Midjourney API | 支持视觉输入的兼容模型 | 支持在设置页自定义配置BaseURL及专属RequestBody格式 |
| **国产替代** | 文心一言/通义千问 | 对应厂商图像生成API | 对应厂商多模态模型 | 国内访问稳定 |

**配置流程**：
1. 首次进入游戏时弹出API Key配置引导
2. 提供图文教程：如何获取API Key
3. 玩家在设置页填入Key并选择模型
4. Key本地加密存储（Hive加密），不上传服务器
5. 支持随时更换Key或切换服务商
6. 支持配置自定义Base URL和请求格式

## 2. 游戏流程设计

```
开始 → 欢迎页 → 创建角色界面 → 选择角色 → 主界面(聊天互动/小游戏) → 好感度变化 → 特殊场景触发 → 好感度达到1000 → 游戏结局
```

### 4.1.3 AI角色外貌生成与图片分析

当用户上传参考图片时，系统通过多模态AI（如 GPT-4o、Claude 3 Opus）分析图片内容，生成详细的外貌描述。

**图片分析Prompt模板**：
```
【任务】
请分析以下参考图片中的人物外貌特征，并生成一份详细的描述，用于AI绘画生成动漫风格的角色形象。

【要求】
1. 详细描述人物的发型、发色、眼睛颜色、脸型、肤色、身材特征
2. 描述人物的穿着风格和常见配饰
3. 描述整体气质和给人的感觉
4. 适合用于日式动漫风格的绘画描述
5. 语言要生动且具体，便于AI绘画理解
6. 字数在150-300字之间

【输出格式】
请直接输出外貌描述文本，无需额外格式。
```

**角色预览图生成Prompt模板**：
```
【角色信息】
角色名：{角色名}
性格：{性格描述}
外貌描述：{AI生成或用户编辑的外貌描述}

【任务】
请生成一张日式动漫风格的角色立绘图片，要求：
1. 风格：精致的日式动漫风格，线条流畅，色彩柔和
2. 角色姿势：自然的站立姿势，面向观众
3. 背景：简单的渐变背景，不要喧宾夺主
4. 分辨率：1024x1024
5. 适合作为游戏角色立绘使用
```

**图片存储要求**：
- 用户上传的参考图片保存在应用沙盒目录 `saves/references/` 下
- 仅在Hive中存储相对路径（如 `/saves/references/photo_001.jpg`）
- 生成的角色预览图保存在 `saves/custom_characters/` 目录下
- 所有图片文件名包含时间戳，避免重名

### 4.2 好感度系统

#### 4.2.1 好感度等级划分（上限1000）

好感度上限提升至1000，每25好感度触发一次AI生成的特殊场景，确保每次游玩体验独特且持久。

| 好感度范围 | 等级名称 | 解锁内容 | 对话风格 |
|------------|----------|----------|----------|
| 0-24 | 陌生人 | 初始对话 | 礼貌、疏远、正式 |
| 25-49 | 初识 | AI生成场景1、专属对话 | 好奇、试探、礼貌 |
| 50-74 | 熟悉 | AI生成场景2、专属对话 | 友好、轻松、关心 |
| 75-99 | 朋友 | AI生成场景3、专属对话 | 亲密、分享、信任 |
| 100-124 | 好朋友 | AI生成场景4、专属对话 | 亲密、关心、默契 |
| 125-149 | 暧昧初期 | AI生成场景5、专属对话 | 暧昧、试探、心跳 |
| 150-174 | 暧昧升温 | AI生成场景6、专属对话 | 暧昧、撒娇、暗示 |
| 175-199 | 告白前夕 | AI生成场景7、专属对话 | 紧张、期待、深情 |
| 200-249 | 恋人初期 | AI生成场景8、专属对话 | 甜蜜、亲昵、热恋 |
| 250-499 | 热恋期 | AI生成场景9-18、专属对话 | 甜蜜、依赖、承诺 |
| 500-749 | 稳定期 | AI生成场景19-28、专属对话 | 温馨、默契、未来规划 |
| 750-999 | 终身承诺 | AI生成场景29-38、专属对话 | 深情、责任、一生约定 |
| 1000 | 永恒结局 | AI生成最终结局 | 永恒、圆满、专属结局 |

**场景触发规则**：
- 每达到25的倍数（25、50、75、100...）时触发一次特殊场景
- 每次场景完全由AI动态生成，内容基于当前对话历史和角色设定
- 场景包括：场景描述（用于生成图片）、剧情对话、互动选项

#### 4.2.2 好感度变化规则

| 对话类型 | 好感度变化 | 示例 |
|----------|------------|------|
| 友好问候 | +2~+5 | "你好呀"、"今天过得怎么样" |
| 关心体贴 | +5~+10 | "注意保暖"、"记得吃饭" |
| 幽默风趣 | +3~+8 | 讲笑话、俏皮话 |
| 深入交流 | +8~+15 | 讨论兴趣爱好、分享心事 |
| 送礼物 | +10~+20 | 根据礼物类型 |
| 小游戏胜利 | +10~+20 | 21点、成语接龙等 |
| 场景互动正确选择 | +15~+30 | 根据选项不同 |
| 负面对话 | -5~-15 | 冷漠、敷衍、冒犯性内容 |
| 小游戏失败 | -5~-10 | 游戏输了 |

#### 4.2.3 好感度上限保护机制

- **每日好感度上限**：每日最多增加100好感度，避免过快通关
- **好感度衰减**：连续7天未登录，好感度每日衰减5%（最多衰减至50%）
- **惩罚机制**：严重负面对话可能导致好感度大幅下降（最高-50）

### 4.3 AI对话系统

#### 4.3.1 对话处理流程

```
用户输入 → 记忆检索 → 意图识别 → 情感分析 → 构建约束Prompt → AI流式生成 → StreamBuilder逐字显示 → 回复校验 → 展示完整回复 → 更新记忆 → 更新好感度
```

**流式传输实现**：
- 使用AI服务的SSE（Server-Sent Events）或WebSocket流式接口
- Flutter端通过StreamBuilder监听数据流
- 每收到一个字符立即显示，实现逐字弹出效果
- 支持打字速度调节（慢/中/快三档）

#### 4.3.2 分层记忆系统

为防止上下文爆炸，采用三层记忆机制：

| 记忆层级 | 存储内容 | 数量限制 | 更新策略 |
|----------|----------|----------|----------|
| **短期记忆** | 最近15条完整对话 | 最多15条 | FIFO，新对话入队，超过15条则移入中期 |
| **中期记忆** | 压缩后的对话摘要 | 最多50条摘要 | 定期合并相似内容，压缩冗余信息 |
| **长期记忆** | 关键信息提炼 | 无限制（但需人工审核） | 手动或自动提炼关键事实写入设定 |

**短期记忆**：
- 存储最近15条完整对话（用户+AI回复配对）
- 保持完整对话内容，不做任何压缩
- 用于维持当前对话上下文连贯性
- 格式：`[时间戳] 用户：XXX\n[时间戳] AI：XXX`

**中期记忆**（压缩机制）：
- 当短期记忆超过15条时，最早的对话移入中期记忆并压缩
- 压缩规则：
  1. 去除重复表达和口头禅
  2. 合并连续相似主题的对话
  3. 提取关键信息（如用户提到的喜好、事件）
  4. 生成2-3句话的摘要
- 示例：
  ```
  原始对话：
  用户：今天天气真好啊
  AI：是啊，阳光明媚的感觉真好
  用户：我最喜欢晴天了
  AI：我也是~ 晴天让人心情愉悦
  
  压缩后：
  用户提到喜欢晴天，双方对天气表示愉悦
  ```

**长期记忆**（提炼机制）：
- 从中期记忆中提炼关键事实，写入角色设定的"共同记忆"字段
- 关键事实包括：
  - 用户明确表达的喜好/厌恶
  - 重要事件（如约会、特殊场景）
  - 建立的昵称和亲密称呼
  - 共同经历的重要时刻
- 示例：
  ```
  长期记忆条目：
  {
    "key": "favorite_weather",
    "value": "用户喜欢晴天",
    "source": "对话记录",
    "timestamp": "2026-05-26"
  }
  ```

**记忆检索优先级**：
1. 首先检索短期记忆（最近15条）
2. 如果短期记忆中没有相关信息，检索中期记忆
3. 如果仍未找到，检索长期记忆（角色设定中的共同记忆）
4. 最后使用默认角色设定

**记忆更新流程**：
```
新对话 → 添加到短期记忆 → 检查短期记忆数量 > 15？ → 是 → 将最早对话压缩后移入中期记忆
                                           ↓
                                    定期（每30分钟）检查中期记忆 → 提炼关键事实 → 写入长期记忆
```

**Prompt中的记忆注入**：
```
【共同记忆】
- 用户喜欢晴天
- 我们上次聊到了你喜欢阅读
- 用户的昵称是"笨蛋"

【近期对话】
[最近3条完整对话]

【用户输入】
{用户当前输入}
```

### 4.4 AI驱动的动态事件与多分支主线生成系统

本系统完全抛弃传统的静态剧本JSON。当`AffectionManager`检测到好感度突破指定阈值（25, 50, 75, 100...）或判定玩家触发了重大的因果转折点时，将暂时冻结主聊天框，无缝切入【全动态事件生成器】。

#### 4.4.1 动态事件技术生成流水线

```
好感度达到阈值 → 记忆检索（分层记忆账本） → AI剧情导演生成场景 → AI生成剧情对话 → AI生成图片Prompt → 并发调用图像引擎 → 渲染全屏CG → 展示场景界面 → 用户选择 → 更新好感度和记忆 → 解锁成就/收藏
```

**场景生成频率**：
- 每25好感度触发一次场景（共40次场景触发机会）
- 第1000好感度为最终结局场景
- 支持因果转折点触发（非好感度驱动的特殊事件）

**核心组件**：

| 组件 | 职责 | 技术实现 |
|------|------|----------|
| **AI剧情导演** | 根据记忆和上下文生成独特剧情 | 调用GPT-4o/GPT-4-turbo |
| **分层记忆账本** | 提供短期/中期/长期记忆数据 | MemoryManager |

#### 4.4.4 动态图片生成机制

**⚠️ 安全存储约束警告**：
> **关键约束**：所有动态生成的CG下载到本地后，Hive数据库中**仅持久化存储其【相对路径】**（如 `/saves/cg/event_25.png`）。UI渲染时再动态拼接当前设备的真实根目录（通过 `getApplicationDocumentsDirectory()` 获取）。
> 
> **原因**：Flutter应用在iOS和Android上的沙盒目录（如 `getApplicationDocumentsDirectory()`）在应用更新时，其绝对路径可能会发生变化（尤其是iOS的UUID目录会变）。如果把DALL-E返回的图片下载后，将绝对路径存进Hive，下次App更新后图片就会全部报404裂图。

**图片存储目录结构**：
```
应用沙盒根目录/
├── saves/
│   └── cg/
│       ├── event_25.png      # 好感度25触发的CG
│       ├── event_50.png      # 好感度50触发的CG
│       └── ending_1000.png   # 结局CG
```

**方案A - 在线图片生成（需联网）**：
- 使用DALL-E 3、Stable Diffusion等API实时生成场景图片
- 根据AI生成的图片提示词调用图片生成API
- 生成后下载并保存到本地 `saves/cg/` 目录
- Hive中仅存储相对路径（如 `/saves/cg/event_25.png`）
- UI渲染时动态拼接：`根目录 + 相对路径`

**方案B - 图片模板合成（纯离线）**：
- 预设多个背景模板和角色姿势
- 根据场景描述选择合适的模板进行组合
- 使用滤镜和特效调整氛围

**方案C - 文字描述替代（极简方案）**：

## 5. AI回答一致性保障机制

### 5.1 角色性格约束系统

#### 5.1.1 性格特征影响规则

| 性格特征 | 参数范围 | 对回答的影响 | 示例 |
|----------|----------|--------------|------|
| **温柔度** | 0-100 | 用词温和、语气柔和、关心体贴 | "没关系的~慢慢来" |
| **活泼度** | 0-100 | 语气轻快、使用表情符号、感叹号 | "好耶！一起去玩吧！🎉" |
| **害羞度** | 0-100 | 用词含蓄、省略号、脸红表情 | "那个...其实我..." |
| **幽默度** | 0-100 | 加入玩笑、俏皮话、双关语 | "你是不是想套路我呀😏" |
| **傲娇度** | 0-100 | 口是心非、别扭表达、反差萌 | "谁...谁要和你一起啊！" |

#### 5.1.2 性格约束执行流程

```
角色数据 → 提取基础性格参数 → 叠加当前情绪值 → 计算动态性格参数 → 构建性格提示词 → 注入AI Prompt → 生成回复
```

**动态性格参数计算**：
- 基础性格参数（来自角色配置）+ 当前情绪值 = 动态性格参数
- 动态参数上限100，下限0，超出范围自动截断
- 情绪随时间衰减，逐渐恢复到基础状态

### 5.2 好感度阶段约束

| +20 | 话题深入 | 开始分享个人事情 |
| +30 | 亲密称呼 | 使用昵称如"笨蛋"、"傻瓜" |
| +50 | 撒娇语气 | 使用"嘛~"、"呢~"等语气词 |

| frequency_penalty | 0.2 | 减少高频词汇重复 |

```
好感度：{具体数值}

【强制要求】
1. 必须严格按照当前性格参数生成回复
2. 语气必须符合当前好感度阶段
3. 禁止生成与历史对话重复的内容
4. 禁止违反角色设定
```

#### 5.4.2 轻量后置校验（滑窗级）

在流式传输过程中进行本地实时检测，避免UI闪烁或内容抹除：

| 校验项 | 校验规则 | 处理方式 |
|--------|----------|----------|
| **性格一致性** | 检查前5个词是否符合角色性格特征 | 不符合则Abort Stream并悄悄重试 |
| **重复检测** | 检测前5个词与历史回复的重复度（相似度>80%） | 高度一致则提前切断流并触发悄悄重试 |
| **内容安全性** | 实时检测敏感词 | 检测到敏感词立即切断流 |
| **回复长度** | 累计字符数超过150则截断 | 截断并添加"..." |

**重复检测算法**：
- 提取流输出的前5个词作为检测窗口
- 与最近10条历史回复进行相似度比对（使用Jaccard相似度）
- 相似度阈值：>80%视为重复
- 触发重复检测后，用户仅看到"正在输入..."时间略微延长，无错误内容展示

**悄悄重试机制**：
1. 检测到问题后立即Abort Stream
2. 保持"正在输入..."状态，后台重新发起请求
3. 重试次数上限：3次
4. 3次重试失败后，显示友好提示并切换到默认回复

#### 5.4.3 完整校验流程

```
用户输入 → 构建带前置约束的Prompt → 发起流式请求
                                          ↓
                              StreamBuilder逐字显示
                                          ↓
                              本地实时检测（滑窗级）
                                          ↓
                              ┌──────────────────────┐
                              │ 检测通过？           │
                              └──────────┬───────────┘
                                         ↓
                              ┌──────────┴───────────┐
                              │                      │
                          是（继续显示）            否（Abort Stream）
                                                      ↓
                                              悄悄重试（最多3次）
                                                      ↓
                                              重试成功？
                                                      ↓
                              ┌──────────┴───────────┐
                              │                      │
                          是（继续显示）            否（显示默认回复）
```

### 5.5 角色情绪系统

#### 5.5.1 情绪状态定义

| 情绪 | 触发条件 | 对回复的影响 |
|------|----------|--------------|
| **开心** | 好感度上升、收到礼物、游戏胜利 | 语气更活泼、使用更多表情 |
| **害羞** | 被夸奖、亲密对话 | 使用省略号、脸红表情 |
| **生气** | 好感度下降、被冒犯 | 语气冷淡、使用感叹号 |
| **难过** | 连续失败、被拒绝 | 语气低沉、使用😢等表情 |
| **撒娇** | 高好感度、请求帮助 | 使用"嘛~"、"呢~"等语气词 |

#### 5.5.2 情绪变化规则

| 事件 | 情绪变化 | 持续时间 | 衰减速度 |
|------|----------|----------|----------|
| 好感度+10 | 开心+20 | 5分钟 | 每分钟-4 |
| 好感度-10 | 难过+15 | 3分钟 | 每分钟-5 |
| 收到赞美 | 害羞+25 | 2分钟 | 每分钟-12.5 |
| 游戏胜利 | 开心+15 | 3分钟 | 每分钟-5 |
| 游戏失败 | 生气+10 | 2分钟 | 每分钟-5 |

**情绪衰减机制**：
- 情绪值随时间线性衰减，直到归零
- 衰减速度 = 初始情绪值 / 持续时间
- 多种情绪可同时存在，叠加影响动态性格参数
- 相同类型情绪可叠加，上限为单次最大值的1.5倍

**情绪对性格参数的影响映射**：
| 情绪 | 影响的性格参数 | 影响方式 |
|------|----------------|----------|
| 开心 | 活泼度+ | 开心值直接加到活泼度 |
| 害羞 | 害羞度+ | 害羞值直接加到害羞度 |
| 生气 | 温柔度-、傲娇度+ | 生气值从温柔度减去，加到傲娇度 |
| 难过 | 活泼度-、温柔度+ | 难过值从活泼度减去，加到温柔度 |
| 撒娇 | 傲娇度+ | 撒娇值加到傲娇度 |

## 6. 互动与共处系统

本系统由两部分组成：**休闲对战小游戏**（保留最具重游价值的 21 点与成语接龙）与**伴学共处系统**（基于番茄工作法的沉浸式陪伴功能），旨在将传统的“对战增进好感”转化为“生活陪伴与共同成长”。

### 6.1 系统分类与触发入口

| 互动类型 | 包含项目 | 核心设计目的 | 好感度边界 |
|:---|:---|:---|:---|
| **休闲对战** | 21点卡牌、成语接龙 | 策略对抗、趣味破冰、高频即时互动 | 胜负触发即时数值起伏 (-4 ~ +7) |
| **伴学共处** | 伴学番茄钟 (Focus Time) | 纯前端生产力工具、长期沉浸式情感陪伴 | 顺利完成触发大额好感度与专注奖励 |

---

### 6.2 休闲对战游戏设计

#### 6.2.1 经典 21 点卡牌 (Blackjack)

* **玩法说明**：玩家与 AI 进行标准 21 点对抗。双方初始两张牌，玩家可自由决定“要牌(Hit)”或“停牌(Stand)”，最接近 21 点且不超过者获胜。
* **AI 动态决策策略**：AI 的停牌阈值不再是固定的 17 点，而是与**角色的动态性格参数/情绪**深度绑定：
    * *基础策略*：当点数 $\ge 17$ 时倾向于停牌。
    * *傲娇度高 / 生气中*：AI 倾向于采取激进策略（点数达到 18 甚至 19 时仍有 30% 概率强行要牌，试图压倒玩家）。
    * *害羞度高 / 温柔度高*：AI 采取极度保守策略（点数达到 15-16 时即选择停牌，避免爆牌）。
* **互动反馈**：AI 爆牌或获胜时，会根据当前情绪向大模型上下文池发送一个快捷标签（如：`[AI_Win_Blackjack]`），在结算弹窗和小游戏结束后的第一句聊天中展现对应的傲娇/温柔台词。

#### 6.2.2 成语接龙 (Idiom Solitaire)

* **玩法说明**：玩家与 AI 轮流输入成语，首尾字需相同（支持同音字首尾相接）。
* **技术实现（纯前端）**：
    * 读取本地 `assets/idioms.json` 成语库进行合法性校验。
    * AI 答题时，在词库中检索匹配字，并根据**角色的幽默度参数**引入“难度扰动”：
        * 幽默度/活泼度高：AI 优先选择带有生僻字或自带趣味意向的成语，且作答延迟更短。
        * 害羞度高：AI 作答时会故意有 1-2 秒的“思考延迟”动画。
* **胜负判定**：限时 30 秒（或无法接出）则输。AI 的成语检索设有上限（如连续接满 15 轮后增加 20% 概率触发“AI词穷”，给玩家获胜空间）。

### 6.3 伴学共处系统 (伴学番茄钟)

将生产力工具番茄钟（Pomodoro）与 AI 角色陪伴融合。玩家学习工作时，AI 角色在屏幕另一端共同专注。

#### 6.3.1 功能流程与界面交互

1.  **专注配置**：
    * 玩家可选择 25 分钟、30 分钟、45 分钟（标准番茄钟）。
    * 可自定义本次专注的任务标签（如：“写代码”、“背德语单词”、“看书”）。
2.  **伴学状态 UI 展示**：
    * **立绘切换**：AI 角色切换为“专注模式”专用立绘或带有配件的图层（例如：戴上黑框眼镜、扎起马尾，桌面摆放一本书）。
    * **环境白噪声**：提供“雨天窗边”、“深夜书房”、“午后咖啡厅”三款环境音效。白噪声的解锁状态与好感度等级、已解锁场景挂钩。
    * **动态粒子效果**：呼吸灯样式的倒计时进度条，UI 整体转为沉浸式、无干扰的高级感冷色调。

#### 6.3.2 防沉迷与切后台惩罚机制（核心逻辑）

为了保障“共同专注”的拟真性，前端必须利用 Flutter 的 `WidgetsBindingObserver` 实时监测 App 状态（`AppLifecycleState`）：

* **轻度分心（防误触缓冲区）**：当玩家手机锁屏，或者 App 切入后台（如去回微信），系统立刻开启一个 **60 秒的“呼唤倒计时”** 并触发手机本地通知：*“林小雨歪了歪头：怎么突然走开了？快回来呀。”*
* **重度失败（放弃/超时）**：若切出后台超过 60 秒，或玩家主动点击“放弃”按钮，番茄钟宣告流产。
    * *系统惩罚*：**即时扣除好感度（根据当前好感度等级扣除 3~8 点）**。
    * *状态影响*：AI 角色触发【难过】或【生气】情绪值（angry/sad +20），立绘变为失落或别扭表情。
    * *AI 台词惩罚*：大模型会在下一次主线聊天中注入提示词，对玩家中途放弃的行为表现出失望或傲娇的鞭策。

#### 6.3.3 顺利完成奖励与“共享休息时间”

* **专注结算奖励**：
    * 好感度大幅度提升：$+\text{专注分钟数} \times 0.4$（例如坚持 45 分钟，好感度 $+18$）。
    * 解锁专属记忆：向中期事实账本（memoryLedger）中增量写入一条事实，记录玩家的专注成就。
* **5分钟共享休息时间（Break Mode）**：
    * 番茄钟完成后，强制进入 5 分钟的白噪音休息时间。
    * 聊天框自动解锁一整套**“伴学专属对话模板”**，AI 会主动递水、揉肩或伸懒腰。
    * 大模型 Prompt 注入：*“你刚刚陪玩家高效完成了‘[任务标签]’任务，现在是 5 分钟休息时间，请主动关心对方，并表现得比平时更亲密一点。”*

### 6.4 互动系统反哺主线对话的“上下文注入机制”

当小游戏或番茄钟结算完毕，玩家关闭结算弹窗、无缝返回主聊天界面时，系统通过临时修改下一次大模型请求的 System Prompt，让 AI 产生完美的“临场即时记忆”。

**执行流程**：
1. 结算完毕后，将游戏结果更新至 `GameSave` 的 `lastGameContextTag` 字段。
2. 下一次发起聊天请求时，在 System Prompt 末尾动态拼接该 Tag 对应的垫片文字。
3. 大模型接收到垫片，第一句话将主动提及刚刚发生的博弈胜负或专注任务。

**临时上下文 Prompt 动态注入示例（以专注失败为例）**：
> `[SYSTEM NOTICE: CURRENT_MINI_GAME_CONTEXT]`
> 玩家刚刚在与你共同专注的番茄钟里玩手机分心，导致共同专注失败（终止前任务标签为：写代码）。你现在心里觉得有些委屈、别扭和失望。请在接下来的流式回复中，第一句话必须以此事切入，用符合你性格特征的傲娇/体贴语气表达你的小情绪，不要直接向玩家生硬朗读本条规则。

- 加密密钥生成方式：首次启动时使用安全随机数生成256位密钥，通过flutter_secure_storage存入平台安全容器（iOS Keychain / Android Keystore）

- Key仅本地存储，不上传任何服务器
- 支持生物识别（指纹/面容）解锁查看Key

**密钥管理流程**：

**说明**：场景和剧情已全部由AI动态生成，无需保留静态场景文件，保持资产包轻量化。

## 8. 技术架构设计

### 8.1 架构分层

```
┌─────────────────────────────────────┐
│           UI Layer                  │
│  (欢迎页、角色选择、聊天界面、场景展示、小游戏) │
├─────────────────────────────────────┤
│         Business Layer              │
│  (角色管理、对话逻辑、好感度计算、场景控制、小游戏逻辑) │
├─────────────────────────────────────┤
│         Data Layer                  │
│  (Hive存储、SharedPreferences、Assets)│
├─────────────────────────────────────┤
│        AI Service Layer             │
│  (对话生成、意图识别、情感分析、游戏AI) │
└─────────────────────────────────────┘
```

### 8.2 核心模块

| 模块 | 职责 | 关键功能 |
|------|------|----------|
| CharacterManager | 角色管理 | 角色数据加载、角色切换 |
| DialogueManager | 对话管理 | 对话发送、AI回复、历史记录 |
| AffectionManager | 好感度管理 | 好感度计算、等级判断 |
| SceneManager | 场景管理 | 场景触发、场景切换 |
| EmotionManager | 情绪管理 | 情绪状态更新、情绪影响 |
| MemoryManager | 记忆管理 | 短期/中期/长期记忆的存储、检索、更新 |
| MiniGameManager | 小游戏管理 | 游戏选择、游戏逻辑、AI策略 |
| AIService | AI服务 | 调用AI模型生成回复 |
| StorageService | 存储服务 | Hive/SharedPreferences操作 |

### 8.3 技术栈

| 分类 | 技术 | 说明 |
|------|------|------|
| 框架 | Flutter | UI开发框架 |
| 状态管理 | Provider/Bloc | 全局状态管理 |
| 本地存储 | Hive | 高性能键值数据库（支持加密） |
| 轻量存储 | SharedPreferences | 简单配置存储 |
| AI流式传输 | SSE / WebSocket | 实现逐字显示效果 |
| 流式UI | StreamBuilder | 实时更新对话气泡 |
| 语音识别 | speech_to_text | 跨平台流式语音转文字，双端稳定，权限处理完善 |
| 安全存储 | flutter_secure_storage | 平台安全容器存储（iOS Keychain / Android Keystore） |
| 图标 | Flutter Icons / Material Design Icons | 图标库 |
| 加密 | Hive AES-256 | API Key加密存储 |

## 9. 交互设计

### 9.1 输入方式

| 输入类型 | 实现方式 | 适用场景 |
|----------|----------|----------|
| 文本输入 | TextField + 发送按钮 | 日常对话 |
| 语音输入 | 语音按钮 + 语音转文字 | 便捷输入 |
| 快捷回复 | 预设选项按钮 | 快速回应 |

### 9.2 反馈设计

| 反馈类型 | 实现方式 | 触发时机 |
|----------|----------|----------|
| 好感度变化 | 数字动画 + 颜色变化 | 每次对话后/小游戏结束后 |
| 特殊场景提示 | 弹窗通知 | 达到好感度阈值 |
| 对话气泡动画 | 渐入效果 | 新消息到达 |
| 角色表情变化 | 立绘切换 | 根据对话情感 |

## 10. 游戏进度与存档

### 10.1 存档内容

| 字段 | 类型 | 说明 |
|------|------|------|
| currentCharacterId | String | 当前选择的角色ID |
| affection | int | 当前好感度值(0-1000) |

| dialogueHistory | List<DialogueRecord> | 对话历史记录 |

- 场景切换时自动保存

- 应用进入后台时自动保存
